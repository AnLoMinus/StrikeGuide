## מדיניות אבטחה ספציפית ל-StrikeGuide / StrikeGuide Specific Security Policy

### דיווח על בעיות אבטחה / Reporting Security Issues

אם גילית בעיית אבטחה, אנא:

1. **אל תפרסם** את הבעיה ב-public
2. **דווח באופן פרטי** דרך:
   - GitHub Security Advisories
   - Issues פרטי (אם אפשר)
3. **תאר בפירוט** את הבעיה
4. **המתן לתגובה** לפני פרסום

### סוגי בעיות אבטחה / Security Issue Types

#### 🔴 קריטי / Critical
- גישה לא מורשית לנתונים
- חשיפת פרטים אישיים
- פגיעות XSS או SQL Injection
- דליפת מידע רגיש

#### 🟡 חשוב / Important
- בעיות הרשאות
- בעיות אימות
- בעיות הצפנה

#### 🟢 נמוך / Low
- בעיות UI/UX
- בעיות ביצועים

### תגובה / Response

אנו מתחייבים:
- ✅ תגובה תוך 48 שעות
- ✅ עדכון תוך שבוע
- ✅ תיקון בהקדם האפשרי
- ✅ קרדיט על מציאת הבעיה (אם מבוקש)

### הגנה על נתונים / Data Protection

המאגר מגן על:
- ✅ פרטיות משתמשים
- ✅ אנונימיות
- ✅ נתונים רגישים
- ✅ מידע אישי

### אמצעי אבטחה / Security Measures

- ✅ GitHub Security Features
- ✅ בדיקות אוטומטיות
- ✅ עדכונים שוטפים
- ✅ בדיקת קוד
